About

Sound Owl web app was built with intention to present our team's ability to conjure, design, build and deploy a web app in two weeks time with features including API integration, AJAX, jQuery, JavaScript and frameworks beyond your basic HTML &andCSS.

The Challenge

    Our team encountered & overcame challenges prior to completion. Luckily, our innovative team navigated them well.

  • API Creation & Deployment. Since this was one of my components, I can speak indepthly on this challenge. I was new to but intrigued with the idea of creating our own API. Having knowledge on API integration, I was able to develop one with LoopBack and Mongo DB. Deployment then became a massive roadblock that faced me with a choice: local tunnel running, where we would be extremely limited to its usage OR I could re-develop the API using PHP, which I am very familiar with, and deploy it like I had my PHP sites in the past. I chose the PHP API method and successfully deployed and integrated it into our app!
  • Time constraint. This project was conjured, designed and built within 2 weeks sprint.
  • Data. Especially in creating the API, a lot of data regarding music and its genres was harvested to make Sound Owl come to life. It is also used via AJAX and XML in the live search feature of the dashboard.
  • Data...BASE. Not something the team was unfamiliar with, but when using and architecting a database, there are usually some issues that're encountered along the way.

Design

Our design team handled the palette and logo design based off or blues with a salmon accent.


I lead the information architecture of Sound Owl to mimic basic dashboard functionality and navigability. Sound Owl runs as a Single-Page Application (SPA) which requires little navigation and more simplicity. Sound Owl is not complex in it's sitemap, there was no need for exaggerated navigation menus or footers.
